## Step 2 — `src/lib/audit.ts`

New file. Shared helper for all client-side audit writes.

```ts
import { addDoc, collection, serverTimestamp } from "firebase/firestore";
import { db } from "@/lib/firebase";

export interface AuditEvent {
  submissionId: string;
  airlineName: string;
  action:
    | "submission.created"
    | "submission.status_changed"
    | "submission.comment_added"
    | "ocr.completed"
    | "ocr.failed";
  actor: { uid: string; email: string; displayName: string };
  metadata: Record<string, unknown>;
}

export async function writeAuditEvent(event: AuditEvent): Promise<void> {
  try {
    await addDoc(collection(db, "audit-log"), {
      ...event,
      timestamp: serverTimestamp(),
    });
  } catch (err) {
    console.error("[audit] Failed to write event:", err);
  }
}
```

Never throws. Caller is never blocked by a failed log write.

**Checkpoint:** Import in browser console, call with a test event, verify document appears in Firestore console.

## Step 3 — `src/lib/exportAudit.ts`

New file. Two export functions, both client-side only.

### Types

```ts
interface AuditEventRecord {
  id: string;
  action: string;
  actor: { email: string; displayName: string };
  timestamp: { seconds: number };
  metadata: Record<string, unknown>;
}

interface SubmissionSummary {
  id: string;
  file_name: string;
  airlineName: string;
  status: string;
  extractedData?: { tailNumber?: string };
}
```

### `exportAuditCSV(submission, events)`

Build comma-separated string with header row:
```
Timestamp,Action,Actor,Details
```

Each event row:
- **Timestamp**: `new Date(e.timestamp.seconds * 1000).toISOString()`
- **Action**: the `action` string verbatim
- **Actor**: `e.actor.email`
- **Details**: `JSON.stringify(e.metadata)` — flatten to single string

Trigger download:
```ts
const blob = new Blob([csv], { type: "text/csv;charset=utf-8;" });
const url = URL.createObjectURL(blob);
const a = document.createElement("a");
a.href = url;
a.download = `audit-${tailNumber}-${date}.csv`;
a.click();
URL.revokeObjectURL(url);
```

### `exportAuditPDF(submission, events)`

Use `jsPDF`:
```ts
import jsPDF from "jspdf";
```

Layout (A4 portrait):
1. **Header block** (top 40mm):
   - Title: "Compliance Audit Report" (bold, 16pt)
   - Submission ID, Tail #, Airline, File name, Current status, Export date (10pt, slate)
2. **Divider line**
3. **Events table** (autoTable-style, manual rows):
   - Columns: Timestamp | Action | Actor | Details
   - Alternate row shading: white / light slate
   - Wrap long Details text to next line if needed
4. **Footer**: "Generated by SkyGate Portal — {date}" (8pt, grey)

Trigger download with `doc.save(filename)`.

**Checkpoint:** Call both functions with mock data, verify CSV opens in Excel and PDF renders correctly.

## Step 4 — Firestore Rules

Add the `audit-log` match block to `firestore.rules` inside the existing `match /databases/{database}/documents` block:

```
match /audit-log/{eventId} {
  allow create: if request.auth != null;
  allow read:   if isAdmin();
  allow update, delete: if false;
}
```

Place after the `fleet-inventory` block.

**Checkpoint:** `firebase deploy --only firestore:rules` succeeds. Test in Firestore emulator or rules playground: unauthenticated write → denied; admin read → allowed; non-admin read → denied; update → denied.

## Step 6 — `src/components/AuditTrail.tsx`

New component. Admin-only audit timeline + export buttons.

```ts
interface AuditTrailProps {
  submissionId: string;
  submission: any; // passed for export header data
}
```

### Internal state

```ts
const [events, setEvents] = useState<AuditEventRecord[]>([]);
const [isLoading, setIsLoading] = useState(true);
const [error, setError] = useState<string | null>(null);
```

### Firestore query

```ts
query(
  collection(db, "audit-log"),
  where("submissionId", "==", submissionId),
  orderBy("timestamp", "desc"),
  limit(50)
)
```

Use `getDocs` (one-time fetch, not `onSnapshot`) — audit logs don't need real-time updates.

### Timeline render

One card per event. Color-coded left dot:
- `submission.created` → slate
- `submission.status_changed` → green (approved) or rose (action required) based on `metadata.to`
- `submission.comment_added` → blue
- `ocr.completed` → purple
- `ocr.failed` → rose

Each card shows:
- **Action label** (human-readable, see mapping below)
- **Actor email** + timestamp (right-aligned)
- **Metadata summary** (one line, formatted per action type)

Action label mapping:
```ts
const ACTION_LABELS: Record<string, string> = {
  "submission.created": "Submission created",
  "submission.status_changed": "Status changed",
  "submission.comment_added": "Comment added",
  "ocr.completed": "OCR extraction completed",
  "ocr.failed": "OCR extraction failed",
};
```

Metadata summary per action:
- `status_changed`: `was {metadata.from} → now {metadata.to}`
- `comment_added`: `"{metadata.commentPreview}"`
- `ocr.completed`: `Tail: {tailNumber} · Confidence: {confidenceScore}% · {complianceStatus}`
- `ocr.failed`: `Error: {metadata.errorMessage}`
- `created`: `{metadata.fileName} ({metadata.fileType})`

### Export buttons

```tsx
<button onClick={() => exportAuditCSV(submission, events)} disabled={events.length === 0}>
  Export CSV
</button>
<button onClick={() => exportAuditPDF(submission, events)} disabled={events.length === 0}>
  Export PDF
</button>
```

Add `title="No audit events recorded yet"` when disabled.

### Error / empty states

- Loading: 3 skeleton rows
- Error: "Audit log unavailable" banner + Retry button (re-runs `getDocs`)
- Empty (no error): "No events recorded yet" with muted subtitle

**Checkpoint:** Render in the drawer with a real submission ID, verify events load and export buttons produce downloadable files.

## Step 8 — Wire Write Points (Cloud Function)

**File:** `functions/src/index.ts`

In `onAirlineUploadOCRTrigger`, after the Gemini extraction block:

**On success** (after `updateDoc` with extracted data):
```ts
await db.collection("audit-log").add({
  submissionId: docId,
  airlineName: extractedData.airlineName || "Unknown",
  action: "ocr.completed",
  actor: { uid: "system", email: "ocr@skygate.aero", displayName: "Gemini OCR" },
  timestamp: admin.firestore.FieldValue.serverTimestamp(),
  metadata: {
    tailNumber: extractedData.tailNumber,
    confidenceScore: extractedData.confidenceScore,
    complianceStatus: extractedData.complianceStatus,
  },
});
```

**On failure** (in the catch block, before `updateDoc` with error status):
```ts
await db.collection("audit-log").add({
  submissionId: docId,
  airlineName: "Unknown",
  action: "ocr.failed",
  actor: { uid: "system", email: "ocr@skygate.aero", displayName: "Gemini OCR" },
  timestamp: admin.firestore.FieldValue.serverTimestamp(),
  metadata: { errorMessage: String(error).slice(0, 500) },
});
```

Wrap each `audit-log` write in its own try/catch — a logging failure must never prevent the function from updating the submission status.

Deploy: `firebase deploy --only functions`

**Checkpoint:** Upload a test file, watch the Audit Trail tab populate with `ocr.completed` event after processing.
